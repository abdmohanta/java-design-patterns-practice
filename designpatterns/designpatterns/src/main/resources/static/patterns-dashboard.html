Pattern Map

Singleton

Short definition: Ensures a class has only one instance and a global access point. Common for configs, caches, or connection pools.

Prototype

Short definition: Create new objects by cloning a prototypical instance—useful when construction is costly or complex.

Factory

Short definition: Encapsulate object creation in a factory method/class to hide concrete types and improve modularity.

Builder

Short definition: Provide a fluent builder API to assemble complex objects step-by-step for readability and immutability.

Abstract Factory

Short definition: Produce families of related objects without specifying their concrete classes; ensures compatibility between products.

Adapter

Short definition: Convert one interface to another so incompatible systems can work together without modifying their code.

Composite

Short definition: Build tree structures of objects so clients can treat individual and composite objects uniformly.

Decorator

Short definition: Dynamically add responsibilities to objects by wrapping them with decorator objects rather than subclassing.

Facade

Short definition: Offer a simplified interface to a complex subsystem, hiding complexity from clients and reducing dependencies.

Proxy

Short definition: Provide a placeholder for another object to control access, lazy-load, or add caching/security layers.

Command

Short definition: Encapsulate requests as objects to support queuing, logging, undo, and decouple senders from receivers.

Observer

Short definition: Maintain dependencies so observers are notified automatically when an observable's state changes—useful for events and pub/sub.

State

Short definition: Allow objects to change behavior when internal state changes by encapsulating states into separate classes.

Strategy

Short definition: Encapsulate interchangeable algorithms behind a common interface so clients can switch strategies at runtime.

Template Method

Short definition: Define an algorithm skeleton in a base class and let subclasses override steps without changing the structure.

Circuit Breaker

Short definition: Temporarily stop calling an unhealthy downstream service, probe it periodically, and resume when it recovers to prevent cascading failures.

Resilience Patterns This directory contains demo implementations for common resilience/fault-tolerance patterns. Files: - `CircuitBreakerPattern.java` - toy circuit breaker local implementation. Expected output: shows state transitions from CLOSED -> OPEN -> HALF_OPEN -> CLOSED depending on failures and open duration. The demo prints calls and failure counts. - `RetryPattern.java` - simple retry policy with exponential backoff and jitter. Expected output: attempts with retry delays and eventual success (or final failure after max attempts). - `TimeoutPattern.java` - uses an ExecutorService to run tasks with a timeout. Expected output: first case completes within timeout; second case times out and is cancelled. - `Resilience4jExamples.java` - a short demo that demonstrates presence of Resilience4j on the classpath. For production examples, the pom includes explicit Resilience4j modules (`resilience4j-core`, `resilience4j-circuitbreaker`, `resilience4j-retry`, `resilience4j-timelimiter`). Notes: - These examples are intentionally minimal and self-contained for learning. They are safe to run locally and print traces to stdout. - For production use, prefer using a mature library like `resilience4j` and integrate it with metrics collection and configuration. Running tests: - The repository includes basic JUnit tests that run each demo's `main` method. Use `mvn test` to run them (requires internet to download test dependencies).

Retry

Short definition: Retry transient operations with configurable backoff and jitter while avoiding overload; ensure idempotency where needed.

Timeout

Short definition: Bound how long you wait for operations, fail fast to free resources, and prevent thread starvation in distributed systems.
